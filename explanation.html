Table

What is a table?

+

Table

The Table is used to store data in a structured format consisting of rows and columns. In Microsoft Access, you can choose between Datasheet View and Design View.


In the Datasheet View, you can look at your Data Field and Data Record. You can add and change your data record in this view, but can't change the properties and format of the data field.

In the Design View, you can add and change the field name, change the data type of the field, and change the properties of the field.


The data types include Short Text, Long Text, Number, Large Number, Date/Time, Currency, Auto Number, and many more.


The properties include Field Size, Format, Input Mask, Caption, Default Value, Required and many more.

Field Size -> The maximum capacity for a text or a specific type of data that the field can hold
Format -> Controls how the data is displayed in the tables, forms, queries, and reports
Input mask -> A template for the characters that gives a specific format for the data in a given field
Caption -> Display user-friendly or easy to understand for the field names in a table
Default Value -> A value that automatically entered in the field when a new record is created
Required -> Forces the user to input value in the given field, doesn't allow null value

Relationship

Relationship?

+

Relationship

Relationship in Microsoft Access is a connection between two or more tables that defines how the data in those tables are related to each other. Creating relationships help ensure data integrity, enforce referential integrity, and to be able to create queries that combine data from multiple tables.


Enforce Referential Integrity -> A rule that ensures relationships between tables stay consistent, accurate, and valid, preventing orphaned records. Enforce Referential Integrity prevents deleting or changing a record that is still related with the original parent record. There are also cascade options: Cascade Update and Cascade Delete.


Cascade Update Related Fields -> A database relational integrity that automatically updates foreign key values in tables when the original primary key in the parent table is changed. To enable cascade update, you must first check Enforce Referential Integrity then check Cascade Update Related Fields.


Cascade Delete Referential Fields -> A database relational integrity that automatically removes a record in a related table when the original parent record is deleted. To enable cascade update, you must first check Enforce Referential Integrity then check Cascade Delete Related Fields.

Form

What is a Form?

+

Form

Form in Microsoft Access is a database object that provides a user-friendly interface for inputting, changing, displaying data from tables or queries. There are 4 ways to create a form in Microsoft Access: Form, Form Design, Form Blank, and Form Wizard.


Form -> Automatically creates a form from the chosen table or query.
Form Design -> Create a form from scratch with a grid template.
Form Blank -> Create a form from scratch with a blank page.
Form Wizard -> Create a customizable form based on your preference from the chosen table or query.


Components in creating a form design or blank form:
Text Box -> Add the field and input in creating a form. Allows users to input new data or change data.
Label -> Add a text in creating a form.
Button -> Add a button for record navigation, record operations, form operations, application control, and many more.
Property Sheet -> To view and edit specific properties in the form.

Query

What is a Query?

+

Query

Query in Microsoft Access is a request for data results and it is used for viewing data, changing, and adding data from one or more tables. There are two methods of creating queries: Query Wizard and Query Design.


Query Wizard -> Create a customizable query based on your own preferences.
Query Design -> Create a query from scratch by choosing data from different tables.


There are several actions in a query such as Query Append, Query Update, Query Crosstab, and Query Delete. To perform these actions, you must press run on the top-left corner in your Microsoft Access.
Query Append -> A query action that retrieves data from one or more tables and adds them to the end of an existing table.
Query Update -> A query action that adds, changes, or deletes data from an existing record in a table.
Query Crosstab -> A query action that calculates a sum, average, or any other aggregate function then groups those results into two values, one in the row heading and one in the column heading.
Query Delete -> A query action that removes specific data from one or more tables based on the given criteria.
Query Make Table -> Creates new tables from data in existing tables or queries.


There are several components in the Query Design Grid in Microsoft Access such as field, table, sort, show, criteria, total function, and parameters.
Field -> Displays the name of the data field in the query column.
Table -> Displays the table of which the data field was taken from.
Sort -> An optional row where you can order how the query results should be displayed.
Criteria -> Enter an expression to filter out the field's data, only showing the results that are suitable for the criteria.
Show -> A checkbox which you can decide whether to display the field's data in the query datasheet or not.
Total -> Only appears when you enable the "Total" features in Access. It is used for grouping records and applying aggregate functions to summarize data.
Parameters -> Used for the users to prompt input when the query runs.

Report

What is a Report?

+

Report

Reports in Microsoft Access are used to view, format, and summarize data from tables or queries. There are a few methods to create a database in Microsoft access such as Report, Report Design, Blank Report, and Report Wizard.


Report -> Automatically creates a report from the chosen table or query.
Report Design -> Create a report from scratch with a grid template.
Blank Report -> Create a report from scratch with a blank page.
Report Wizard -> Create a customizable report based on your preference from the chosen table or query.
